Detailed Description

Definitions related to inverse semigroups of partial permutations.

This file defines several auxiliary functions used behind the scenes of PartialPermInverseSemigroup. All functions and their implementations are based on [east16].

Compute the component of the action of a set of partial permutations (the generators) on a set of elements \(\alpha\) as well an associated orbit graph and spanning Schreier tree.

The action component results from the repeated application of the generators to \(\alpha\) (application refers to the operation performed by the PartialPerm::image() function) and all resulting sets of elements until a fixpoint is reached. The resulting orbit graph is the unique directed graph connecting nodes corresponding to the resulting sets with edges labelled with the generators that produce the destination node's set when applied to the source node's set. The Schreier tree is a spanning tree through the orbit graph rooted at the node corresponding to \(\alpha\) (in most cases there are several possible such spanning trees, this function makes no guarantees about which one is returned). Orbit graph and Schreier tree can be treated as opaque by code which uses the functions defined in this header.

Parameters
alphaa set of elements in the form of a vector, the first element in the resulting action component and root of the resulting orbit graph and Schreier tree
generatorsa set of partial permutations in the form of a vector
dom_maxthe maximum over the values PartialPerm::dom_max() of all generators (passed as an argument to avoid frequent recalculation)
[out]schreier_treethe resulting Schreier tree
[out]orbit_graphthe resulting orbit graph
Returns
the resulting action component in form of a vector of vectors; Schreier tree and orbit graph are internally represented using indices into this vector and are thus only meaningful in combination with it

std::vector< PartialPerm > const &  generators

Calculate the \(\mathscr{R}\)-class respresentatives of an inverse partial permutation semigroup from a corresponding Schreier tree.

See [east16] for details on the theory begin \(\mathscr{R}\)-classes. In the special case of an inverse semigroup of partial permutations \(G\) the \(\mathscr{R}\)-class representatives can be determined by first obtaining a Schreier tree for on orbit graph constructed using a generating set for the group and rooted at \(\cup_{g \in G} dom(g)\) and then calculating the partial permutations that result if for every path from the Schreier tree's root to one its leaves the corresponding partial permutation edge labels are chained together.

Parameters
schreier_treethe Schreier tree rooted at \(\cup_{g \in G} dom(g)\) which is traced in order to determine the \(\mathscr{R}\)-class representatives
generatorsthe partial permutations which constitute the Schreier tree's edge labels
Returns
a vector containing the \(\mathscr{R}\)-class representatives in no particular order

std::pair< unsigned, std::vector< unsigned > > mpsym::internal::eemp::strongly_connected_components ( OrbitGraph const &  orbit_graph)

Find the strongly connected components of an orbit graph.

The strongly connected components (short s.c.c.'s) are a partition of the orbit graph such that all nodes in a strongly connected components can be directly or indirectly reached from every other node in the strongly connected component.

Parameters
orbit_graphorbit graph for which the s.c.c.'s are to be determined
Returns
a pair in which the first element is the number of resulting s.c.c's and the second element is a vector containing as many elements as there are nodes in the orbit graph (and thus elements in the corresponding action component) in which two elements have the same value if and only if the nodes corresponding to their indices in the vector lie in the same s.c.c.

Example:

Assume that an action component \(\{\alpha_1, \alpha_2, \alpha_3, \alpha_4\}\) is partitioned according to the s.c.c.'s of an associated orbit graph as follows: \(\{\{\alpha_1\}, \{\alpha_2, \alpha_3\}, \{\alpha_4\}\}\). This function might then return {3u, {0u, 1u, 1u, 2u}} (the concrete values assigned to each s.c.c. in the resulting vector are not guaranteed).
