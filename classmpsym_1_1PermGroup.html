Detailed Description

A permutation group representation.

This class provides a useful abstraction encapsulating several complex algorithms and data structures used to efficiently represent a permutation group defined by a set of generating permutations without the need to store elements explicitely for very large groups.

mpsym::PermGroup::PermGroup ( unsigned  degree,
PermSet const &  generators

Construct a permutation group.

Constructs a permutation group representation from a given set of generating permutations. The generators and group elements might not be stored explicitly in the resulting object. Instead some variation of the Schreier-Sims algorithm (see [holt05], chapter 4.4.2) might be used to compute a base and strong generating set for the group which describes the group completely and can be used to, among others, test element membership and iterate through all group elements efficiently.

Parameters
degreethe permutation group's degree, must be the same as the degree of all given generators (which in turn implies that they map from the set \(\{1, \dots, degree\}\) to itself) otherwise this constructor's behaviour is undefined
generatorsa generating set for the permutation group

const_iterator mpsym::PermGroup::begin ( ) const
inline

Obtain a constant iterator iterating over this group's elements.

Note that the permutation group elements might not be stored explicitly and could instead be constructed by the iterator "on the fly" which means that storing references or pointers to the partial permutation pointed to by any iterator iterating over a permutation group will result in undefined behaviour. The elements are not guaranteed to be returned in any particular order but every element will be returned exactly once.The elements will also be returned in the same order on subsequent iterations through the permutation group. This last point is not necessarily true for other equal (in the sense of operator==) permutation groups.

Iterating through a permutation group could thus look like this:

PermGroup pg(degree, generators);
for (auto const &perm : pg)
std::cout << perm;
for (auto const it = pg.begin(); it != pg.end(); ++i)
std::cout << *it;
Returns
a constant iterator pointing to some element in this permutation group, incrementing it will yield all other group members exactly once (in no particular order) until end() is reached.

std::vector< PermGroup > mpsym::PermGroup::disjoint_decomposition ( bool  complete = true,
bool  disjoint_orbit_optimization = false 
) const

Find a disjoint subgroup decomposition of a permutation group.

This function's implementation is based on [donaldson09].

A disjoint subgroup decomposition of a permutation group \(G\) is a set \(\{H_1, \dots, H_k\}\) where \(H_1, \dots, H_k\) are subgroups of \(G\) such that \(G = \{\alpha_1 \cdot \alpha_2 \dots \alpha_k : \alpha_i \in H_i, (1 \leq i \leq k)\}\) and \(\forall (i, j) \in \{1, \dots, k\}^2 : i \neq j \Rightarrow moved(H_i) \cap moved(H_j) = \emptyset\). Here, \(moved(H_i)\) is the set \(\{x : (x \in \{1, \dots, n\} \land \exists \alpha \in H_i : \alpha(x) \neq x)\}\).

Parameters
completeif this is true the function will always return the finest possible disjoint subgroup decomposition (i.e. one in which no subgroup can be further decomposed into disjoint subgroups), this might be more computationally expensive
disjoint_orbit_optimizationif this is true, the optimization described in [donaldson09], chapter 5.2.1 is applied, this is only meaningful if complete = true, otherwise this argument is ignored.
Returns
a disjoint subgroup decomposition of this permutation group given as a vector of subgroups as described above, in no particular order, if no disjoint subgroup partition could be found, the vector contains the group itself as its only element.

std::vector< PermGroup > mpsym::PermGroup::wreath_decomposition ( ) const

Find a wreath product decomposition of a permutation group.

This function's implementation is based on [donaldson09].

A wreath product decomposition of a permutation group \(G \leq S_n\) is (using the notation in [donaldson09]) a triple \((H, K, \mathcal{X})\). Here \(H \leq S_m\), \(K \leq S_d\) and \(n = m \cdot d\), \(\mathcal{X}\) is a set \(\{X_1, \dots, X_d\}\) of sets \(X_i\) (with \(|X_i| = m\)) which partition the set \(X = \{1, \dots, n\}\) on which \(G\) operates and \(G = \{\sigma(\beta)\ \sigma_1(\alpha_1) \dots \sigma_d(\alpha_d) : \beta \in K, \alpha_i \in H \ (1 \leq i \leq d)\}\). \(\sigma\) is the permutation representation of the action of \(K\) on \(\mathcal{X}\) which permutes the sets \(X_i\) "among themselves" and the \(\sigma_i\) are the permutation representations of the obvious actions of \(K\) on the sets \(X_i\). This is conventionally written as: \(G = H \wr K\).

Returns
a wreath product decomposition of \(G\) as described above in the form of the vector of permutation groups \([\sigma(K), \sigma_1(H_1), \dots, \sigma_d(H_d)]\), if no wreath product decomposition could be found (either because none exists or the algorithm is unable to determine a valid decomposition, which is currently possible due to limitations in the implementation) an empty vector is returned
